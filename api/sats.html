Send and receive satoshis

note

By default the calls are directed to the staging environment where the funds are worthless, using the default bitcoin signet.

The GraphQL endpoint to connect to:
The following methods require a valid auth token set in the header as a bearer token:

Get the wallet IDs and check the balances

Can run this query at any stage to confirm the change in the balances.
The "BTC" wallet balance is denominated in satoshis.
The "USD" wallet balance is in cents.
The body of the GraphQL request:
  query Me {
    me {
      defaultAccount {
        wallets {
          id
          walletCurrency
          balance
        }
      }
    }
  }

Generate an invoice
Receive satoshis to the BTC balance.
Set the variables
The body of the GraphQL request:
mutation LnInvoiceCreate($input: LnInvoiceCreateInput!) {
  lnInvoiceCreate(input: $input) {
    invoice {
      paymentRequest
      paymentHash
      paymentSecret
      satoshis
    }
    errors {
      message
    }
  }
}
curl command to generate an invoice:
Probe invoice fee
Estimate the cost of paying a lightning invoice.
Payments to an other Blink user and to nodes with a direct channel are free.
Set the variables
The body of the GraphQL request:
mutation lnInvoiceFeeProbe($input: LnInvoiceFeeProbeInput!) {
  lnInvoiceFeeProbe(input: $input) {
    errors {
      message
    }
    amount
  }
}
cURL command to probe invoice fee:
Pay an invoice
Pay a BOLT11 invoice from your BTC balance
Set the variables
The body of the GraphQL request:
mutation LnInvoicePaymentSend($input: LnInvoicePaymentInput!) {
  lnInvoicePaymentSend(input: $input) {
    status

      message

      code
    }
  }
